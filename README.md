# Лабораторная работа №5. Дерево Выражений

Проект реализует дерево арифметических выражений на C++17. Дерево можно строить из постфиксной или инфиксной записи, выводить выражение в разных формах, вычислять результат, упрощать константные подвыражения и работать с переменными.

# Основные возможности

* построение дерева выражения из постфиксной записи;
* построение дерева выражения из инфиксной записи;
* преобразование инфиксной записи в постфиксную;
* вычисление значения выражения;
* вывод выражения в инфиксной форме;
* вывод выражения в префиксной форме;
* вывод выражения в постфиксной форме;
* поддержка операторов +, -, *, /, ^;
* поддержка отрицательных чисел;
* поддержка переменных;
* хранение переменных в собственной map-структуре на AVL-дереве;
* подсчёт высоты дерева;
* подсчёт количества операторов;
* упрощение константных подвыражений;
* обработка ошибок вычисления через исключения.

## Основные компоненты

### ExprNode

Базовый абстрактный класс узла дерева выражения.

Определяет общий интерфейс для всех узлов:

* вычисление значения;
* вывод в инфиксной, префиксной и постфиксной форме;
* подсчёт высоты;
* подсчёт количества операторов;
* проверка, является ли подвыражение константным;
* упрощение подвыражения.

### NumberNode

Узел числовой константы.

Хранит целое число и при вычислении возвращает его значение.

### VariableNode

Узел переменной.

Хранит имя переменной. При вычислении получает значение переменной из хранилища VariableStorage.

Значения переменных хранятся как строки, после чего при вычислении преобразуются в целое число через std::stoi.

Если переменная не задана, выбрасывается исключение std::logic_error.

Если значение переменной нельзя преобразовать в целое число, также выбрасывается std::logic_error.

### BinaryOperatorNode

Узел бинарной операции.

Хранит оператор и два поддерева: левый и правый операнд.

Поддерживаемые операторы:

* "+"
* "-"
* "*"
* "/"
* "^"

При делении на ноль выбрасывается исключение std::logic_error.

Если оператор неизвестен, также выбрасывается std::logic_error.

### ExpressionTree

Основной класс дерева выражений.

Отвечает за:

* построение дерева;
* преобразование инфиксной записи в постфиксную;
* вычисление выражения;
* вывод выражения;
* работу с переменными;
* упрощение выражения;
* очистку дерева.

### AVLTree

Собственная реализация ассоциативного контейнера на AVL-дереве.

Используется для хранения переменных выражения.

В проекте применяется шаблонная реализация AVLTree<Key, Value> и специализация для <std::string, std::string>.

Для строковой версии дополнительно реализован поиск по подстроке через алгоритм Кнута — Морриса — Пратта (Префикс-Функция).

## Пример использования

ExpressionTree tree;

std::vector[std::string](std::string) expr = {
"(", "a", "+", "b", ")", "*", "2"
};

tree.build_from_infix(expr);

tree.setVariable("a", "3");
tree.setVariable("b", "5");

int result;
tree.evaluate(result);

std::cout << result << std::endl;

Результат:

16

Примеры выражений

Постфиксная запись:

2 3 + 4 5 * +

Соответствующее выражение:

(2 + 3) + (4 * 5)

Инфиксная запись:

(2 + 3) * 4

После преобразования в постфиксную запись:

2 3 + 4 *

Упрощение выражений

Если подвыражение состоит только из констант, оно может быть заменено одним числовым узлом.

Например:

(2 + 3) * x

После упрощения:

5 * x

Переменные при упрощении не вычисляются как константы.

## Обработка ошибок

Проект использует исключения std::logic_error для ошибок вычисления.

Исключение выбрасывается в следующих случаях:

* попытка вычислить пустое дерево;
* деление на ноль;
* обращение к неопределённой переменной;
* значение переменной не является корректным целым числом;
* значение переменной выходит за диапазон int;
* встречен неизвестный оператор.

## Особенности реализации

* дерево выражений построено через полиморфизм;
* память узлов дерева управляется через std::unique_ptr;
* переменные хранятся в собственной AVL-map;
* AVL-map реализована как шаблонный контейнер;
* для AVLTree<std::string, std::string> добавлена специализация со строковыми методами;
* инфиксная запись преобразуется в постфиксную, после чего строится дерево;
* вычисление выражения выполняется рекурсивно.

## Ограничения

* выражение должно быть токенизировано заранее: каждый оператор, число, переменная и скобка передаются отдельной строкой;
* унарный минус поддерживается для чисел вида -5, но не как отдельный оператор перед переменной;
* переменные при вычислении должны содержать строку, которую можно преобразовать в int;
* оператор ^ вычисляется через std::pow с приведением результата к int.
